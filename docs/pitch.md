# ohmcp 作品亮点（答辩提纲）

## 一句话

为 OpenHarmony 重造 MCP 的"底盘"：保持 MCP 语义不变，把 JSON-RPC 文本传输
替换为二进制帧 + 多路复用 + 内容寻址缓存 + 默认加密的原生协议栈——
线上字节最高 **−99.8%**（共享内存通道），吞吐最高 **+84%**，p99 尾延迟最高 **−67%**，
六大场景吞吐全部为正（每场景 3 次取中位数）。

## 对照赛题评审维度

### 技术完备性（30%）

- 7 crate 模块化 Rust workspace，全部可编译、可运行、可复现；
- 39 项单元测试 + 3 项端到端集成测试（SHM 并发正确性/大小混合回退/路径一致性）（含对抗性：篡改/截断/超限帧/随机字节流不 panic）+ 端到端多 Agent 演示 + 六场景基准（每场景 3 次取中位数）；
- 行覆盖率 83.7%（llvm-cov，安全模块 100%、帧编解码 96.6%、软总线适配 92.9%）；
- CI：fmt + clippy(-D warnings，零警告) + 全量测试 + 基准烟雾；
- 生产级细节：帧大小上限、协议版本协商、AEAD 防篡改（帧头入 AAD）、
  30s 请求超时、错误码规范（-32601/-32001/-32002）。

### 开源合规（20%）

- Apache-2.0；全部源码原创，第三方仅 crates.io 依赖引用（0% 代码拷贝）；
- 连续、语义化的提交历史（feat/perf/docs/chore 分层）。

### 场景适配（25%）

- 端侧多 Agent 复用单守护进程（UDS），单连接 64 路并发无队头阻塞；
- 字节效率即能耗：面向软总线/BLE/低功耗链路的 −81%~−95.6% 传输量；
- 软总线集成设计 + PoC 代码：DSoftBus Session 适配器（`softbus.rs`）已落地，
  帧/压缩/加密层零改动运行于消息导向会话之上（设计文档 §7.1）。

### 创新性（25%）

1. **CACHE_REF 内容寻址缓存**：服务端/客户端对称缓存，热点重复调用仅传
   32 字节哈希（−94% 字节，+37%~+59% 吞吐）——MCP 生态中未见同类机制；
2. **机会主义内联读**：顺序调用零任务切换、并发调用自动变身 combiner，
   兼得同步 RPC 的低延迟与异步多路复用的高吞吐；
3. **默认安全且可量化**：HMAC 挑战应答（令牌不过网）+ 每会话 ChaCha20-Poly1305
   （单调计数 nonce、原地加密），加密开销在报告中逐场景量化；
4. **共享内存大 payload 通道**：memfd 密封环形缓冲 + SCM_RIGHTS fd 传递，
   同设备 64KB 结果零套接字拷贝（帧内仅 12 字节引用），吞吐 +52%、
   套接字字节 −99.8%，空间不足自动回退——对齐软总线“同设备快速通道”理念。
